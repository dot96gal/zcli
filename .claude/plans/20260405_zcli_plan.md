# zcli: Zig CLI Framework — 実装計画レポート

**前提条件:**
- Zig 標準ライブラリのみ使用（サードパーティ依存なし）
- `comptime` を活用した型安全・ゼロコスト抽象化
- フラグ解析をフレームワーク内に含める
- Google `subcommands`（Go）に近いインタフェース
- テスト容易性のため stdout/stderr/allocator を `Env` 経由で注入可能にする

---

## コア型

### ExitStatus (`exit_status.zig`)

```zig
pub const ExitStatus = enum(u8) {
    success     = 0,
    failure     = 1,
    usage_error = 2,

    pub fn toCode(self: ExitStatus) u8 {
        return @intFromEnum(self);
    }

    pub fn exit(self: ExitStatus) noreturn {
        std.process.exit(self.toCode());
    }
};
```

### Command (`command.zig`)

```zig
pub const Command = struct {
    ptr: *anyopaque,
    vtable: *const VTable,

    pub const VTable = struct {
        name:     *const fn () []const u8,
        synopsis: *const fn () []const u8,
        usage:    *const fn (*anyopaque, *std.Io.Writer) std.Io.Writer.Error!void,
        run:      *const fn (*anyopaque, []const []const u8, *Env) anyerror!ExitStatus,
    };

    // デリゲートメソッド
    pub fn name(self: Command) []const u8 { ... }
    pub fn synopsis(self: Command) []const u8 { ... }
    pub fn usage(self: Command, w: *std.Io.Writer) !void { ... }
    pub fn run(self: Command, args: []const []const u8, env: *Env) !ExitStatus { ... }

    // comptimeでT型からvtableを自動生成
    pub fn from(comptime T: type, ptr: *T) Command { ... }
};
```

`name` と `synopsis` はコンパイル時定数のためselfポインタ不要。
`run` は `anyerror!ExitStatus` で、コマンドの自由度を最大化する。
`Env` を受け取ることで stdout/stderr/allocator をテストから差し替え可能にする。

### FlagSet (`flag_set.zig`)

```zig
pub const FlagType = union(enum) {
    string: struct { default: []const u8 },
    bool:   struct { default: bool        },
    int:    struct { default: i64         },
};

pub const FlagDef = struct {
    long:        []const u8,   // "verbose"
    short:       ?u8,          // 'v' または null
    flag_type:   FlagType,
    description: []const u8,
};

/// 内部型。利用者は getString/getBool/getInt 経由でアクセスするため直接使用しないこと。
const FlagValue = union(enum) {
    string: []const u8,
    bool:   bool,
    int:    i64,
};

pub const ParseError = error{
    UnknownFlag,
    MissingValue,
    InvalidIntValue,
    OutOfMemory,
};

pub const FlagSet = struct {
    allocator:       std.mem.Allocator,
    defs:            []const FlagDef,                   // comptimeスライス（ヒープ不使用）
    values:          std.StringHashMap(FlagValue),       // キー・値ともにdupe()で所有
    /// 内部フィールド。直接アクセスせず positionals() メソッドを使うこと。
    positionals_buf: std.ArrayList([]const u8),

    pub fn init(allocator: std.mem.Allocator, comptime defs: []const FlagDef) FlagSet
    pub fn deinit(self: *FlagSet) void                   // キー・値・positionals_buf をすべて解放
    pub fn parse(self: *FlagSet, args: []const []const u8) ParseError!void
    pub fn positionals(self: *const FlagSet) []const []const u8  // positionals_buf.items を返す
    pub fn getString(self: *const FlagSet, name: []const u8) ?[]const u8
    pub fn getBool(self: *const FlagSet, name: []const u8) ?bool
    pub fn getInt(self: *const FlagSet, name: []const u8) ?i64
};
```

- `defs` は `comptime` 既知スライス → バイナリのデータセグメントに配置、ヒープ不使用
- `values` のキー・値は `allocator.dupe()` でコピー所有 → `args` の寿命に依存しない
- `positionals_buf` は内部フィールド（非公開）。公開メソッド `positionals()` が `.items` を返す
  - フィールドとメソッドの同名衝突（Zigのコンパイルエラー）を避けるための命名
- `parse()` は `void` を返す（位置引数は `positionals()` で取得）
- `deinit()` でキー・値・`positionals_buf` をすべて解放する
- フラグ定義のlookupは線形スキャン（通常 < 20件のため十分）

### Env / TestEnv (`env.zig`)

```zig
pub const Env = struct {
    allocator: std.mem.Allocator,
    stdout:    std.io.AnyWriter,
    stderr:    std.io.AnyWriter,

    /// 本番用: 実際の stdout/stderr を使う Env を生成する
    pub fn init(allocator: std.mem.Allocator) Env {
        return .{
            .allocator = allocator,
            .stdout    = std.io.getStdOut().writer().any(),
            .stderr    = std.io.getStdErr().writer().any(),
        };
    }
};

/// テスト用ヘルパー。stdout/stderr をバッファに向けた Env を提供する。
pub const TestEnv = struct {
    out_buf:   std.ArrayList(u8),
    err_buf:   std.ArrayList(u8),
    allocator: std.mem.Allocator,

    pub fn init(allocator: std.mem.Allocator) TestEnv {
        return .{
            .out_buf   = std.ArrayList(u8).init(allocator),
            .err_buf   = std.ArrayList(u8).init(allocator),
            .allocator = allocator,
        };
    }

    /// *self 経由でライターを生成するため、self が生きている限り常に有効なポインタを持つ。
    /// 呼び出し後に self を移動させてはならない。
    pub fn env(self: *TestEnv) Env {
        return .{
            .allocator = self.allocator,
            .stdout    = self.out_buf.writer().any(),
            .stderr    = self.err_buf.writer().any(),
        };
    }

    pub fn deinit(self: *TestEnv) void {
        self.out_buf.deinit();
        self.err_buf.deinit();
    }
};
```

- `Env` は stdout/stderr/allocator を保持するが、それらを所有はしない
- `TestEnv` は `env` フィールドを持たず、`env()` メソッドで毎回 `*self` から生成する
  - `init()` が値返却時にポインタがずれるダングリングバグを回避する
- ユーザーは `var env = te.env()` で取得した `Env` を `*Env` として渡す

### Commander (`commander.zig`)

```zig
pub const Commander = struct {
    env:         Env,                    // 値でコピー所有（*Env ではない）
    program:     []const u8,
    description: []const u8,
    commands:    std.ArrayList(Command),

    pub fn init(env: Env, program: []const u8, description: []const u8) Commander
    pub fn deinit(self: *Commander) void
    /// コマンドインスタンスの寿命は Commander より長くなければならない。
    /// Commander はポインタを保持するが所有権は持たない。
    pub fn register(self: *Commander, cmd: Command) !void
    pub fn run(self: *Commander, args: []const []const u8) !ExitStatus
};
```

- `Commander` は `Env` を値でコピーして保持する
  - `Env` の中身（`std.mem.Allocator`, `std.io.AnyWriter`）はいずれも「ポインタ+vtable」の軽量な値型なのでコピーは安全
  - `*Env` と違い、元の `Env` の寿命に依存しないため寿命管理の暗黙的な制約がなくなる
- `allocator` は `env.allocator` 経由でアクセスする（重複フィールドを避ける）
- `register()` はポインタを保存するのみ。コマンドインスタンスの所有権は持たない

## comptime 活用ポイント

### 1. `validateCommand` — コンパイル時インタフェース検証（非公開）

```zig
// pub なし: command.zig 内プライベート
/// Command.from() から呼ばれる内部検証関数。直接呼び出し不要。
fn validateCommand(comptime T: type) void {
    const ti = @typeInfo(T);
    if (ti != .@"struct")
        @compileError("Command type must be a struct, got: " ++ @typeName(T));

    if (!@hasDecl(T, "name"))
        @compileError(@typeName(T) ++ " must declare: pub fn name() []const u8");
    if (!@hasDecl(T, "synopsis"))
        @compileError(@typeName(T) ++ " must declare: pub fn synopsis() []const u8");
    if (!@hasDecl(T, "usage"))
        @compileError(@typeName(T) ++ " must declare: pub fn usage(*T, *std.Io.Writer) !void");
    if (!@hasDecl(T, "run"))
        @compileError(@typeName(T) ++ " must declare: pub fn run(*T, []const []const u8, *Env) !ExitStatus");
}
```

- `@hasDecl`, `@typeInfo`, `@TypeOf`, `@compileError` を使用
- 実行時コスト: ゼロ（コンパイル時に解決）
- `Command.from(T, ptr)` 内から呼ばれる実装詳細のため非公開
  - ユーザーは `Command.from()` を呼ぶだけでよく、`validateCommand` を意識する必要がない
  - `pub` にすると公開 API のノイズになるため除外

### 2. `Command.from` — vtable自動生成

```zig
pub fn from(comptime T: type, ptr: *T) Command {
    comptime validateCommand(T);

    // T型ごとに1つのvtableをデータセグメントに配置
    const vtable = comptime &VTable{
        .name = struct {
            fn f() []const u8 { return T.name(); }
        }.f,

        .synopsis = struct {
            fn f() []const u8 { return T.synopsis(); }
        }.f,

        .usage = struct {
            fn f(p: *anyopaque, w: *std.Io.Writer) std.Io.Writer.Error!void {
                return @as(*T, @ptrCast(@alignCast(p))).usage(w);
            }
        }.f,

        .run = struct {
            fn f(p: *anyopaque, args: []const []const u8, env: *Env) anyerror!ExitStatus {
                return @as(*T, @ptrCast(@alignCast(p))).run(args, env);
            }
        }.f,
    };

    return .{ .ptr = ptr, .vtable = vtable };
}
```

- `comptime &VTable{...}` → T型ごとに1インスタンス、読み取り専用データセグメントに配置
- `@ptrCast(@alignCast(...))` → `*anyopaque` から具体的ポインタを復元する標準イディオム

### 3. `comptime defs` — フラグ定義のゼロコスト配置

```zig
// FlagSet.init の comptime defs パラメータ
pub fn init(allocator: std.mem.Allocator, comptime defs: []const FlagDef) FlagSet {
    return .{
        .allocator       = allocator,
        .defs            = defs,    // ヒープ不使用: バイナリのデータセグメントへのスライス
        .values          = std.StringHashMap(FlagValue).init(allocator),
        .positionals_buf = std.ArrayList([]const u8).init(allocator),
    };
}
```

### 4. `inline for` — ヘルプ生成の静的最適化

```zig
// help.zig
pub fn printFlagHelp(w: *std.Io.Writer, comptime defs: []const FlagDef) !void {
    inline for (defs) |def| {
        // 各イテレーションでflag_typeのブランチをコンパイル時解決
        const default_str: []const u8 = switch (def.flag_type) {
            .string => |s| s.default,
            .bool   => |b| if (b.default) "true" else "false",
            .int    => |i| comptime std.fmt.comptimePrint("{d}", .{i.default}),
        };
        try w.print("  --{s}", .{def.long});
        if (def.short) |sh| try w.print(", -{c}", .{sh});
        try w.print("\n        {s} (default: {s})\n", .{ def.description, default_str });
    }
}
```

- `inline for` + `comptime defs` → 到達不能ブランチを除去
- 整数デフォルト値は `std.fmt.comptimePrint` でコンパイル時文字列化

## フラグ解析アルゴリズム

`FlagSet.parse(args)` はサブコマンド名より後のスライスを受け取る。
位置引数は `FlagSet` 内部に保持し、`positionals()` で取得する（所有権を一元管理）。

```
入力: args = argv[サブコマンド名の次以降]
出力: void（位置引数は fs.positionals() で取得）

i = 0
while i < args.len:
    arg = args[i]

    if arg == "--":
        残り全部を位置引数に追加して終了

    if arg が "--" で始まる:
        body = arg[2..]
        bool フラグの場合:
            "--verbose" → store("verbose", true)  ※ 値トークンを消費しない
        '=' を含む場合（非boolフラグのみ）:
            key = body[0..eq], value = body[eq+1..]  →  store(key, value)
        その他（非boolフラグ）:
            次のargをvalueとして消費  →  store(key, args[i+1])

    if arg が "-X" 形式（len==2）:
        short_char で FlagDef を検索
        bool フラグ → store(long_name, true)  ※ 値トークンを消費しない
        その他 → 次のargをvalueとして消費

    else:
        位置引数として追加

// 未設定フラグにデフォルト値を挿入
for each def in self.defs:
    if not in values: insert default
// 位置引数は self.positionals_buf に保持（toOwnedSlice しない）
```

**文字列所有権のルール:**
- フラグのキー・値はすべて `allocator.dupe()` でコピーして `FlagSet` が所有する
- `args` の寿命が `FlagSet` より短くなっても安全
- `deinit()` でキー・値・`positionals_buf` をすべて解放する

### boolフラグの動作規則

bool フラグはプレゼンスフラグとして扱う。値トークンは消費しない。

| 指定方法 | 結果 |
|---|---|
| `--verbose` が存在する | `true` |
| `--verbose` が存在しない | `FlagDef` のデフォルト値（通常 `false`） |

`--verbose true` や `--verbose false` はサポートしない（`true`/`false` は後続の位置引数として扱われる）。

## Commanderルーティング

```zig
pub fn run(self: *Commander, args: []const []const u8) !ExitStatus {
    const argv = if (args.len > 0) args[1..] else args;  // argv[0]（プログラム名）をスキップ

    if (argv.len == 0) {
        try self.printTopLevelHelp();
        return .usage_error;
    }

    const subcmd_name = argv[0];

    // 組み込みhelpコマンド
    if (std.mem.eql(u8, subcmd_name, "help")) {
        return self.handleHelp(argv[1..]);
    }

    // 登録済みコマンドを検索
    for (self.commands.items) |cmd| {
        if (std.mem.eql(u8, cmd.name(), subcmd_name)) {
            return cmd.run(argv[1..], &self.env);  // &self.env でポインタを渡す
        }
    }

    // 未知コマンド（stderr は self.env.stderr 経由）
    try self.env.stderr.print("unknown command: {s}\n\n", .{subcmd_name});
    try self.printTopLevelHelp();
    return .usage_error;
}
```

### 組み込みhelpコマンド

| 呼び出し | 動作 |
|---|---|
| `mytool help` | 全コマンド一覧と synopsis を表示 |
| `mytool help greet` | `greet` コマンドの `usage()` を呼ぶ |
| `mytool help unknown` | エラーメッセージ + `failure` |

## 公開APIと利用例

### コマンド実装

```zig
// example/greet_command.zig
const std = @import("std");
const zcli = @import("zcli");

const greet_flags = [_]zcli.FlagDef{
    .{
        .long        = "name",
        .short       = 'n',
        .flag_type   = .{ .string = .{ .default = "World" } },
        .description = "Name to greet",
    },
    .{
        .long        = "count",
        .short       = 'c',
        .flag_type   = .{ .int = .{ .default = 1 } },
        .description = "Number of times to greet",
    },
    .{
        .long        = "excited",
        .short       = 'e',
        .flag_type   = .{ .bool = .{ .default = false } },
        .description = "Use exclamation mark",
    },
};

pub const GreetCommand = struct {
    pub fn name() []const u8 { return "greet"; }
    pub fn synopsis() []const u8 { return "print a greeting message"; }

    pub fn usage(_: *GreetCommand, w: *std.Io.Writer) !void {
        try w.print("usage: <program> greet [flags]\n\nFlags:\n", .{});
        try zcli.help.printFlagHelp(w, &greet_flags);
    }

    pub fn run(_: *GreetCommand, args: []const []const u8, env: *zcli.Env) !zcli.ExitStatus {
        // allocator は env 経由で取得（コマンド構造体に持つ必要がない）
        var fs = zcli.FlagSet.init(env.allocator, &greet_flags);
        defer fs.deinit(); // キー・値・positionals_buf をすべて解放

        fs.parse(args) catch |err| {
            try env.stderr.print("flag error: {s}\n", .{@errorName(err)});
            return .usage_error;
        };

        const target  = fs.getString("name")  orelse "World";
        const count   = fs.getInt("count")    orelse 1;
        const excited = fs.getBool("excited") orelse false;
        const punct: u8 = if (excited) '!' else '.';

        // stdout は env 経由 → テストで差し替え可能
        var i: i64 = 0;
        while (i < count) : (i += 1) {
            try env.stdout.print("Hello, {s}{c}\n", .{ target, punct });
        }
        return .success;
    }
};
```

### main.zig

```zig
const std = @import("std");
const zcli = @import("zcli");
const GreetCommand = @import("greet_command.zig").GreetCommand;

pub fn main() !void {
    var gpa = std.heap.DebugAllocator(.{}){};
    defer _ = gpa.deinit();
    const allocator = gpa.allocator();

    // std.process.argsAlloc は [][:0]u8 を返す
    const raw_args = try std.process.argsAlloc(allocator);
    defer std.process.argsFree(allocator, raw_args);
    // [:0]u8 → []const u8 のcoercingを明示。Commander.run は []const []const u8 を受け取る
    const args: []const []const u8 = raw_args;

    // Env は値で Commander に渡す（*Env の寿命管理が不要）
    var cmdr = zcli.Commander.init(zcli.Env.init(allocator), "mytool", "A demonstration CLI tool");
    defer cmdr.deinit();

    var greet = GreetCommand{};
    try cmdr.register(zcli.Command.from(GreetCommand, &greet));

    const status = cmdr.run(args) catch |err| blk: {
        std.debug.print("error: {s}\n", .{@errorName(err)});
        break :blk zcli.ExitStatus.failure;
    };
    status.exit();
}
```

## 主要設計判断と理由

### vtable + `*anyopaque` vs tagged union
tagged unionは全コマンド型をコンパイル時に既知にする必要があり、拡張性を破壊する。vtableはオープン拡張モデルを維持しつつ、ディスパッチコスト（1間接参照）はCLIの用途では無視できる。

### `comptime defs: []const FlagDef`
フラグ定義はコマンドごとに不変。comptimeにすることで:
- 定義配列のヒープ確保ゼロ
- `inline for` による到達不能ブランチ除去
- 整数デフォルトの `comptimePrint` 対応

### `StringHashMap` for values、線形スキャン for defs lookup
値はparse時に設定し複数回読まれる → O(1)のHashMapが適切。
定義は1回walkするだけ・通常20件未満 → 線形スキャンで十分、余分なHashMap不要。

### `[]const []const u8` for `Commander.run` の引数
OS依存の型（`[][:0]u8`）をフレームワーク内部に持ち込まない。テストでは文字列リテラルのスライスをそのまま渡せる。本番では `std.process.argsAlloc` で取得した `[][:0]u8` を `[]const []const u8` に変換して渡す（`[:0]u8` → `[]const u8` は暗黙的にcoerce可能）。

### `anyerror!ExitStatus` for `run`
コマンドがI/O・アロケーション・任意の処理を行えるよう自由度を最大化。エラーセットを制限するとコマンドごとに変換が必要になる。`Commander`境界でcatchして診断メッセージ+`failure`に変換する。

### bool フラグはプレゼンスフラグ
`--verbose true` / `--verbose false` のような値トークンを受け付ける設計は、`true`/`false` を文字列として解析する処理と `InvalidBoolValue` エラーが必要になり実装が複雑になる。CLI の慣習として「フラグの存在 = true」が自然であり、デフォルト値は `FlagDef` で管理すれば十分。値トークンを消費しないためパーサーも単純になる。

### DuplicateFlag は返さない（後勝ち）
同一フラグを複数回指定した場合にエラーにすると、スクリプトからフラグを重ねて渡すユースケース（`base_flags + override_flags` のような合成）で使いにくくなる。多くの CLI パーサーの慣習である「後勝ち（last wins）」を採用し、`ParseError` からも除外することで実装を簡潔に保つ。

## メモリ管理上の注意点

### FlagSet: 文字列は dupe() でコピー所有
`parse()` 内でフラグのキー・値を `allocator.dupe()` でコピーし、`FlagSet` が所有権を持つ。
`args` スライスが `FlagSet` より先に解放されても安全。`deinit()` で全文字列を解放する。

### FlagSet: 位置引数は内部 `positionals_buf` で管理
`parse()` は `void` を返し、位置引数は `positionals()` メソッド（`positionals_buf.items` を返す）で取得する。
呼び出し側が個別に `free()` する必要がなく、所有権が `FlagSet` に一元化される。
フィールド名を `positionals_buf` にすることで、同名メソッド `positionals()` との衝突を回避している。

### Commander: コマンドポインタの寿命
`Commander` は `Command`（`ptr: *anyopaque`）のポインタを保持するが所有権は持たない。
コマンドインスタンス（`var greet = GreetCommand{}`）は `Commander` より長寿命でなければならない。

```zig
// NG: greet が Commander より先にスコープを抜ける
{
    var greet = GreetCommand{};
    try cmdr.register(zcli.Command.from(GreetCommand, &greet));
} // ← greet 解放。cmdr.run() 時にダングリング

// OK: 同じスコープで寿命を揃える
var greet = GreetCommand{};
try cmdr.register(zcli.Command.from(GreetCommand, &greet));
cmdr.run(args) catch {};
```

### Env: ライター寿命は TestEnv.env() で保証

`Env` の `stdout`/`stderr` は保持するが所有しない。
テスト時は `TestEnv.env()` で `*self` 経由のライターを毎回生成することでダングリングを防ぐ。

```zig
// NG: TestEnv が env フィールドを持ち init() で設定する旧設計
// init() がスタック上の self から writer を作り値返却すると
// env.stdout が解放済みスタックを指すダングリングになる

// OK: TestEnv.env() メソッドを使う
var te = zcli.TestEnv.init(testing.allocator);
defer te.deinit();
var env = te.env();   // *te から生成 → te が生きている限り有効
                      // env 取得後に te を移動・再代入してはならない
_ = try cmd.run(&.{"--name", "Alice"}, &env);
try testing.expectEqualStrings("Hello, Alice.\n", te.out_buf.items);
```

// src/hello_command.zig
const std = @import("std");
const zcli = @import("zcli");

pub const HelloCommand = struct {
    pub fn name() []const u8 { return "hello"; }
    pub fn synopsis() []const u8 { return "say hello"; }

#### パターン B: `--name` フラグで名前を受け取る

```zig
// src/hello_command.zig
const std = @import("std");
const zcli = @import("zcli");

const hello_flags = [_]zcli.FlagDef{
    .{
        .long        = "name",
        .short       = 'n',
        .flag_type   = .{ .string = .{ .default = "world" } },
        .description = "Name to greet",
    },
};

pub const HelloCommand = struct {
    pub fn name() []const u8 { return "hello"; }
    pub fn synopsis() []const u8 { return "say hello"; }

    pub fn usage(_: *HelloCommand, w: *std.Io.Writer) !void {
        try w.print("usage: <program> hello [flags]\n\nFlags:\n", .{});
        try zcli.help.printFlagHelp(w, &hello_flags);
    }

    pub fn run(_: *HelloCommand, args: []const []const u8, env: *zcli.Env) !zcli.ExitStatus {
        var fs = zcli.FlagSet.init(env.allocator, &hello_flags);
        defer fs.deinit();

        fs.parse(args) catch |err| {
            try env.stderr.print("flag error: {s}\n", .{@errorName(err)});
            return .usage_error;
        };

        const target = fs.getString("name") orelse "world";
        try env.stdout.print("Hello, {s}!\n", .{target});
        return .success;
    }
};
```

実行例:
```
$ my_tool hello --name Alice
Hello, Alice!
$ my_tool hello -n Alice
Hello, Alice!
$ my_tool hello
Hello, world!
```

```zig
// src/main.zig
const std = @import("std");
const zcli = @import("zcli");
const HelloCommand = @import("hello_command.zig").HelloCommand;

pub fn main() !void {
    var gpa = std.heap.DebugAllocator(.{}){};
    defer _ = gpa.deinit();
    const allocator = gpa.allocator();

    const raw_args = try std.process.argsAlloc(allocator);
    defer std.process.argsFree(allocator, raw_args);
    const args: []const []const u8 = raw_args;

    var cmdr = zcli.Commander.init(zcli.Env.init(allocator), "my_tool", "My CLI tool");
    defer cmdr.deinit();

    var hello = HelloCommand{};
    try cmdr.register(zcli.Command.from(HelloCommand, &hello));

    const status = cmdr.run(args) catch |err| blk: {
        std.debug.print("error: {s}\n", .{@errorName(err)});
        break :blk zcli.ExitStatus.failure;
    };
    status.exit();
}
```

### 5. 公開APIリファレンス

#### `Command`

| シンボル | シグネチャ | 説明 |
|---|---|---|
| `Command.from` | `fn (comptime T: type, ptr: *T) Command` | 任意の型からコマンドを生成。`T` が必要なメソッドを持つかをコンパイル時に検証 |
| `Command.name` | `fn (Command) []const u8` | コマンド名を返す |
| `Command.synopsis` | `fn (Command) []const u8` | 一行説明を返す |
| `Command.run` | `fn (Command, []const []const u8, *Env) anyerror!ExitStatus` | コマンドを実行する |

`T` が実装すべきメソッド:

```zig
pub fn name() []const u8 { ... }                                         // selfなし
pub fn synopsis() []const u8 { ... }                                     // selfなし
pub fn usage(*T, *std.Io.Writer) !void { ... }
pub fn run(*T, []const []const u8, *zcli.Env) !zcli.ExitStatus { ... }
```

#### `FlagSet`

| メソッド | シグネチャ | 説明 |
|---|---|---|
| `FlagSet.init` | `fn (std.mem.Allocator, comptime []const FlagDef) FlagSet` | フラグセットを初期化 |
| `FlagSet.deinit` | `fn (*FlagSet) void` | 所有メモリをすべて解放 |
| `FlagSet.parse` | `fn (*FlagSet, []const []const u8) ParseError!void` | 引数を解析してフラグと位置引数を格納 |
| `FlagSet.positionals` | `fn (*const FlagSet) []const []const u8` | 位置引数スライスを返す |
| `FlagSet.getString` | `fn (*const FlagSet, []const u8) ?[]const u8` | 文字列フラグの値を取得 |
| `FlagSet.getBool` | `fn (*const FlagSet, []const u8) ?bool` | boolフラグの値を取得 |
| `FlagSet.getInt` | `fn (*const FlagSet, []const u8) ?i64` | 整数フラグの値を取得 |

#### `FlagDef`

```zig
pub const FlagDef = struct {
    long:        []const u8,   // フラグ名（例: "verbose"）
    short:       ?u8,          // 省略形（例: 'v'）。不要なら null
    flag_type:   FlagType,     // .string / .bool / .int とデフォルト値
    description: []const u8,   // ヘルプに表示する説明
};
```

#### `Commander`

| メソッド | シグネチャ | 説明 |
|---|---|---|
| `Commander.init` | `fn (Env, []const u8, []const u8) Commander` | プログラム名・説明を設定して初期化 |
| `Commander.deinit` | `fn (*Commander) void` | コマンドリストを解放 |
| `Commander.register` | `fn (*Commander, Command) !void` | コマンドを登録 |
| `Commander.run` | `fn (*Commander, []const []const u8) !ExitStatus` | argv[0] をスキップしてサブコマンドをルーティング |

#### `Env` / `TestEnv`

| シンボル | シグネチャ | 説明 |
|---|---|---|
| `Env.init` | `fn (std.mem.Allocator) Env` | 本番用: 実際の stdout/stderr を使う `Env` を生成 |
| `TestEnv.init` | `fn (std.mem.Allocator) TestEnv` | テスト用: バッファに向けた `Env` を提供 |
| `TestEnv.env` | `fn (*TestEnv) Env` | `*self` 経由で `Env` を生成（呼び出し後に `*TestEnv` を移動禁止） |
| `TestEnv.deinit` | `fn (*TestEnv) void` | `out_buf` / `err_buf` を解放 |

#### `ExitStatus`

| 値 | コード | 説明 |
|---|---|---|
| `.success` | 0 | 正常終了 |
| `.failure` | 1 | エラー終了 |
| `.usage_error` | 2 | 使い方のエラー |

`status.exit()` でプロセスを該当コードで終了する。

### 6. メモリ管理のポイント（利用者向け要約）

- **`FlagSet`**: `deinit()` を必ず呼ぶ。`defer fs.deinit()` 推奨
- **`Commander`**: `deinit()` を必ず呼ぶ。登録したコマンドの実体（`GreetCommand` など）は Commander より長寿命にすること
- **`TestEnv`**: `deinit()` を必ず呼ぶ。`env()` で取得した `Env` の使用中は `TestEnv` を移動させないこと
- **`Env`**: 所有しているリソースはなし。`deinit()` 不要

## 振り返り：計画と実装の差分

### 計画から変更した点

#### 1. `Env` のフィールド型と `Env.init()` の廃止

- **計画**: `stdout: std.io.AnyWriter`, `stderr: std.io.AnyWriter`、`Env.init(allocator: std.mem.Allocator) Env` あり
- **実装**: `stdout: *std.Io.Writer`, `stderr: *std.Io.Writer`、`Env.init()` なし

Zig 0.15.2 では `std.io.AnyWriter` は deprecated。新しい `std.Io.Writer` は vtable の drain 関数が `@fieldParentPtr("interface", io_w)` で親構造体を復元する設計のため、値コピーが安全でない。そのため `Env` は `*std.Io.Writer` を保持し、backing の writer（`std.fs.File.Writer` など）の寿命管理は呼び出し元（`main`）が担う設計に変更した。

`main` での使用例:
```zig
var stdout_buf: [4096]u8 = undefined;
var stderr_buf: [512]u8 = undefined;
var stdout_w = std.fs.File.stdout().writer(&stdout_buf);
var stderr_w = std.fs.File.stderr().writer(&stderr_buf);
// writer が cmdr より先に宣言され寿命を包む

var cmdr = zcli.Commander.init(
    zcli.Env{
        .allocator = allocator,
        .stdout = &stdout_w.interface,
        .stderr = &stderr_w.interface,
    },
    "mytool", "A demonstration CLI tool",
);
```

#### 2. `TestEnv` の内部バッファ型の変更

- **計画**: `out_buf: std.ArrayList(u8)` / `err_buf: std.ArrayList(u8)` → `.writer().any()` で `AnyWriter`
- **実装**: `out_w: std.Io.Writer.Allocating` / `err_w: std.Io.Writer.Allocating`

`std.ArrayList(u8).writer()` も Zig 0.15.2 では deprecated（"Deprecated in favor of `print` or `std.io.Writer.Allocating`"）。`std.Io.Writer.Allocating` は flush が no-op でデータをバッファに蓄積するため、テストでの出力確認に適している。出力確認は `te.out_w.writer.buffered()` で行う（計画の `te.out_buf.items` に相当）。

#### 3. writer 引数型の統一

- **計画**: `usage()` の引数 `w` は `*std.Io.Writer`（一部コードで `std.io.AnyWriter` と混在）
- **実装**: VTable・公開メソッド・ユーザー実装コードすべて `*std.Io.Writer` に統一

#### 4. `std.ArrayList` の API 変更対応

- **計画**: managed API（`.init(allocator)`, `.append(item)`, `.deinit()`）
- **実装**: unmanaged API（`.empty`, `.append(allocator, item)`, `.deinit(allocator)`）

Zig 0.15.2 で `std.ArrayList(T)` が unmanaged に変更（アロケータをフィールドに持たず、各操作にアロケータを渡す）。`StringHashMap` は managed のまま変更なし。

#### 5. `example/main.zig` での明示的 `flush`

- **計画**: `status.exit()` を呼ぶだけ
- **実装**: `status.exit()` 前に `stdout_w.interface.flush()`・`stderr_w.interface.flush()` を明示

`ExitStatus.exit()` が `std.process.exit()` を呼ぶためデファーがスキップされ、バッファの内容が OS に書き出されない。`exit()` 直前に明示的にフラッシュする必要がある。

#### 6. `example/build.zig` 未作成

- **計画のファイル構成**: `example/build.zig`（サンプル単体のビルド定義）を用意
- **実装**: `example/build.zig` なし。メインの `build.zig` にサンプルのビルドを統合

### 計画になかった追加事項

#### 7. `example/greet_command.zig` の分離

- **計画のファイル構成**: `example/main.zig` のみ記載
- **実装**: `example/greet_command.zig` を別ファイルとして分離し、`main.zig` から `@import` する形に

#### 8. `build.zig` への `run` ステップ追加

- **計画の `build.zig`**: `run` ステップなし
- **実装**: `run` ステップを追加（`mise run run` および `mise run example` から利用）

#### 9. `mise.toml` への `example` タスク追加

- **計画**: `build`・`test`・`run` の3タスクのみ
- **実装**: `example` タスクを追加（`zig build run --summary all -- $@`）。`--` で zig build への引数とバイナリへの引数を分離し、`$@` でユーザー引数を転送する。
